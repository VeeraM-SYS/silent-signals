The following is a set of guidelines for contributing to Silent Signals. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## Code Style

- We use **Dart's official style guide**.
- **Riverpod** is our state management solution.
- **clean architecture** principles are encouraged (keep logic out of UI).
- Ensure your code is lint-free: `flutter analyze`.

## How to Submit Changes

1. **Fork the Repo**: Create your own fork of the code.
2. **Create a Branch**: `git checkout -b feature/amazing-feature`
3. **Commit**: `git commit -m 'Add some amazing feature'`
4. **Push**: `git push origin feature/amazing-feature`
5. **Open a Pull Request**: Go to the repo on GitHub and click "Compare & pull request".

## Reporting Bugs

This section guides you through submitting bug reports.

- **Use a clear and descriptive title**.
- **Describe the reproduction steps** in detail.
- **Include screenshots** if the issue is visual.
